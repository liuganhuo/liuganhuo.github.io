条款20 宁以pass-by-reference-to-const替换pass-by-value

C++在调用函数传递参数的时候

对于自定义类型的类，如果直接通过值传递，则会对应地发生copy过程，整个类连带父类的各个成员都会进行拷贝，并在函数调用结束时再次调用析构。这会极大地提高成本。

因此可以考虑在传递参数时采用引用的形式，没有任何构造函数和析构函数被调用，

但是在使用引用传递参数时，注意设置参数为const类型，确保函数内不会对传入的类做出改变

这样做还可以避免**对象切割问题(slicing)**。当一个子类对象通过值传递方式传递，但是函数声明时，声明使用父类对象，则copy函数对调用父类的copy函数，进而产生一个父类对象而非子类对象。

当然，对于内置类型或者STL类型的对象来说，选择值传递的方式则更好
